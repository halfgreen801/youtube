# 개골튜브

개골튜브는 YouTube 계정과 분리해서 영상과 숏츠 링크를 저장하는 개인용 PWA입니다. 앱 주소는 공개될 수 있지만, 기본 저장 데이터는 현재 기기의 브라우저 `localStorage` 안에만 저장됩니다. Supabase 동기화를 직접 설정하고 로그인한 경우에만 같은 계정의 클라우드 데이터와 동기화됩니다.

## 이 앱은 무엇인가

- YouTube 영상/숏츠 링크 저장
- 카테고리와 분류 칸 관리
- 메모, 검색, 상태 필터, 즐겨찾기
- 앱 안에서 YouTube 임베드 재생
- JSON 내보내기/가져오기 백업
- 카테고리별 공유용 JSON 내보내기
- 기기별 파스텔 색상 설정
- GitHub Pages에 올릴 수 있는 정적 PWA

## 기본 저장 방식

기본값은 `localStorage`입니다.

- 같은 브라우저/같은 기기에서는 저장 데이터가 유지됩니다.
- 같은 GitHub Pages 주소로 다른 기기에서 접속해도 자동으로 보이지 않습니다.
- 브라우저 데이터 삭제, 시크릿 모드 종료, 저장소 초기화가 발생하면 로컬 데이터가 사라질 수 있습니다.
- 중요한 데이터는 `내보내기`로 JSON 백업을 만들어 두세요.

## GitHub Pages 배포 방법

1. 이 저장소를 GitHub에 올립니다.
2. GitHub 저장소의 `Settings > Pages`로 이동합니다.
3. 배포 소스를 `Deploy from a branch`로 선택합니다.
4. 브랜치와 폴더를 선택합니다. 보통 `main` / `/root`입니다.
5. Pages 주소로 접속해서 앱이 열리는지 확인합니다.

정적 파일만 사용하므로 별도 백엔드가 필요 없습니다.

## 앱 설치와 색상 설정

- `앱 설치` 버튼은 지원 브라우저에서만 보입니다.
- 앱 설치는 홈 화면 또는 앱 목록에 추가하는 기능입니다. 저장 목록 동기화 기능이 아닙니다.
- iPhone/iPad에서는 설치 팝업이 자동으로 뜨지 않습니다. Safari 공유 버튼에서 `홈 화면에 추가`를 직접 선택해야 합니다.
- iPhone에서는 홈 화면 앱과 Safari 탭의 저장공간이 분리될 수 있습니다. 목록이 한쪽에만 보이면 `데이터 이전`에서 백업 파일을 만든 뒤 다른 쪽에서 가져오세요.
- 여러 기기에서 같은 목록을 쓰려면 JSON `내보내기`/`가져오기` 또는 Supabase 동기화를 사용하세요.
- `색상 설정`에서 배경색, 카드색, 글자색, 강조색 등을 기기별로 바꿀 수 있습니다.
- 색상 설정은 `gaegol-tube-theme-v1` localStorage 키에 저장되며, 영상 목록 데이터와 분리됩니다.
- 색상 설정은 다른 기기로 자동 동기화되지 않습니다.

## iPhone 홈 화면 앱과 Safari 저장공간

iPhone에서는 같은 GitHub Pages 주소라도 Safari 탭에서 연 화면과 홈 화면에 추가한 앱의 `localStorage`가 분리될 수 있습니다.

- Safari에서 목록이 비어 보여도 삭제된 것이 아니라 홈 화면 앱 쪽 저장공간에 남아 있을 수 있습니다.
- 홈 화면 앱을 삭제하기 전에 반드시 `데이터 이전` 또는 `내보내기`로 JSON 백업 파일을 만들어 두세요.
- 새 링크, Safari, 다른 기기에서 같은 목록을 보려면 목록이 보이는 쪽에서 백업 파일을 만든 뒤 목록이 없는 쪽에서 `가져오기`를 사용합니다.
- 자동 동기화를 원하면 Supabase 같은 클라우드 동기화를 직접 설정해야 합니다.
- Supabase를 처음 사용할 때는 목록이 있는 기기에서 먼저 `현재 기기 데이터 업로드`를 누르고, 목록이 없는 기기에서는 그 다음 `클라우드 데이터 불러오기`를 누르세요.

## 여러 기기 동기화 사용 방법

동기화는 선택 기능입니다. 설정하지 않아도 앱은 localStorage-only 모드로 정상 작동합니다.

1. Supabase 프로젝트를 만듭니다.
2. `Authentication`에서 이메일/비밀번호 로그인을 켭니다.
3. 개인용으로 쓸 계정을 먼저 만든 뒤, 공개 회원가입을 끄는 방식을 권장합니다.
4. Supabase SQL Editor에서 `supabase/schema.sql` 내용을 실행합니다.
5. `config.example.js`를 참고해 `config.js`를 수정합니다.

```js
window.TUBE_VAULT_CONFIG = {
  syncEnabled: true,
  supabaseUrl: "https://YOUR_PROJECT.supabase.co",
  supabaseAnonKey: "YOUR_SUPABASE_ANON_PUBLIC_KEY",
  allowSignup: false
};
```

6. GitHub Pages에 다시 배포합니다.
7. 앱의 `저장 방식` 영역에서 로그인합니다.
8. 클라우드 데이터가 있으면 `클라우드 데이터 불러오기`, `현재 기기 데이터 업로드`, `가능하면 병합` 중 하나를 선택합니다.

`allowSignup`이 `false`이면 앱 안의 회원가입 버튼은 숨겨집니다. 개인용 앱이라면 Supabase 대시보드에서 내 계정을 만든 뒤 `allowSignup: false`로 두는 구성이 안전합니다.

## 보안 주의사항

- `service_role` key, 관리자 키, 개인 토큰은 절대 브라우저 코드에 넣지 마세요.
- 브라우저에는 Supabase `anon/public` key만 들어갈 수 있습니다.
- 저장소가 public이면 앱 코드와 `config.js`의 anon key는 공개될 수 있습니다.
- anon key는 공개 가능하므로, RLS 없이 쓰면 위험합니다.
- 데이터 보호의 핵심은 `supabase/schema.sql`의 Row Level Security입니다.
- `public.tube_vault_states`는 `auth.uid()`가 `user_id`와 같은 행만 읽고 쓸 수 있어야 합니다.

## 백업/복원 방법

- `내보내기`: 현재 저장된 링크, 카테고리, 분류 칸을 JSON 파일로 저장합니다.
- `데이터 이전`: 현재 실행공간의 모드, 주소, 저장 키, 항목 수를 확인하고 이전용 백업 파일을 만듭니다.
- `가져오기`: JSON 파일의 항목을 현재 기기 데이터에 병합합니다. 이미 있는 같은 영상/숏츠는 건너뜁니다.
- 새 내보내기 파일명은 `gaegol-tube-YYYY-MM-DD.json` 형식입니다. 기존 `tube-vault` 파일도 계속 가져올 수 있습니다.
- 데이터 이전 백업 파일명은 `gaegol-tube-migration-YYYY-MM-DD.json` 형식입니다.
- 클라우드 데이터를 불러오기 전에는 앱이 자동으로 `tubeVaultBackupBeforeCloudPull:<timestamp>` localStorage 백업을 만듭니다.
- Supabase 동기화를 처음 켜기 전에는 수동 JSON 내보내기를 한 번 해두는 것을 권장합니다.

## 카테고리 공유 방법

전체 백업이 아니라 특정 카테고리의 목록만 다른 사람에게 보내고 싶을 때 사용합니다.

1. 상단의 `카테고리 공유`를 누릅니다.
2. 공유할 카테고리를 선택합니다.
3. 포함할 정보, 예를 들어 메모/상태/즐겨찾기 여부를 선택합니다.
4. `공유 파일 만들기`를 누릅니다.
5. 생성된 JSON 파일을 상대방에게 보냅니다.
6. 상대방은 `가져오기`로 파일을 불러옵니다.

- 공유 파일은 선택한 카테고리의 목록만 포함합니다.
- 메모, 봤음 상태, 즐겨찾기는 기본적으로 제외됩니다.
- `분류 칸 포함`은 기본으로 켜져 있으며, 끄면 모든 항목이 `기본` 칸으로 들어갑니다.
- 공유받은 사람의 앱에 같은 이름의 카테고리가 있으면 그 카테고리에 병합됩니다.
- 자동 동기화 공유 링크가 아니라 JSON 파일을 직접 전달하는 방식입니다.

## 수동 테스트 체크리스트

- 새 방문자가 앱을 열면 localStorage-only 모드로 정상 작동한다.
- 앱 제목과 manifest name/short_name이 `개골튜브`로 보인다.
- 기존 localStorage 데이터가 유지된다.
- iPhone 홈 화면 앱과 Safari 저장공간 분리 안내가 보인다.
- `데이터 이전`에서 현재 실행 모드, origin, 저장 키, 항목 수, 앱 버전이 표시된다.
- `데이터 이전`에서 현재 목록 백업 파일을 만들 수 있다.
- 모바일 360px 폭에서 레이아웃이 깨지지 않는다.
- 링크 추가/수정/삭제가 정상 작동한다.
- export/import가 정상 작동한다.
- `카테고리 공유`에서 선택한 카테고리의 항목만 JSON으로 내보낼 수 있다.
- 공유 파일을 `가져오기`로 불러오면 같은 이름의 카테고리에 병합되고 중복 영상은 추가되지 않는다.
- 색상 설정에서 색상을 바꾸면 즉시 UI가 바뀐다.
- 새로고침 후에도 저장한 색상이 유지된다.
- 초기화 버튼을 누르면 기본 파스텔 테마로 돌아간다.
- Supabase 설정값이 비어 있어도 console fatal error가 없다.
- Supabase 설정 후 로그인/로그아웃이 된다.
- 로그인 후 수동 동기화가 된다.
- 다른 기기 또는 다른 브라우저에서 같은 계정으로 로그인하면 클라우드 데이터를 불러올 수 있다.
- 오프라인 상태에서도 로컬 저장은 된다.
- service worker 캐시 업데이트 후 새 UI가 반영된다.
